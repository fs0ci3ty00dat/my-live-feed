# CONTEXT.md — Where This Data Goes

## Background

I build mini-apps on a mobile platform called Wabi (wabi.ai). It's a React Native environment where apps are generated by AI and run natively on iOS. Think of it as a personal app store where you can create, publish, and share small focused apps.

I've been building on the platform for a few months and have around 80 published apps — utilities, games, data tools, visual experiences.

---

## The Problem This Repo Solves

Wabi apps have one way to pull external data: a `web_search` tool that works like a browser search. It's great for general content but unreliable for structured live data like sports scores or news headlines — the results are noisy, inconsistent, and sometimes cached.

There's no support for raw `fetch()`, `axios`, or direct API calls. No websockets, no background fetch.

This site is my workaround. By serving clean, structured HTML at a stable URL, I can point the `web_search` call at my own page and get reliable, predictable data every time.

---

## How Wabi Uses This Site

Inside a Wabi app prompt, I tell the AI agent to fetch from a specific URL instead of doing a general search:

```
Use web_search to fetch data from:
https://tomeranderson.github.io/my-live-feed/scores.html

Parse the HTML and extract the score lines.
```

The Wabi AI reads the HTML exactly like a browser would — it scrapes the visible text content. That's why the HTML must be clean, minimal, and human-readable with no JavaScript rendering.

## Platform Constraints to Keep in Mind

These affect how the HTML should be structured:

- The Wabi web_search reads rendered text content — not raw JSON, not JavaScript-rendered content
- It works best on **concise pages under ~50 lines of visible text**
- Timestamps matter — Wabi's AI can infer freshness from a "Last updated:" line
- Simpler = better: h1/h2/p tags, plain text scores, no tables
